## ShokaX 的现状

现在是 2025年12月13日，ShokaX Astro 迁移刚刚进入非生产 Alpha 版本，这是继：

- valaxy-theme-shokaX（2023年11月10日-2024年2月14日，未产出任何成果便终止开发）
- astro-theme-shokaX（2024年2月15日-2024年6月24日，未产出任何成果便终止开发）
- Hexo ShokaX SMB （ShokaX 0.3/0.4/0.5，事实上的缝合怪）
- ShokaX UIKit（2025年1月27日-2025年12月12日，产出了部分 Vue3 组件库和 Nyx-Player，经转译用于 ShokaX Astro）
  后，首个在现代博客平台上推出的基本可用的 ShokaX 迁移方案

在连续废了4个项目后，ShokaX 总算有产出了，虽然走的弯路疑似有点多了

本文系统性梳理一下这四个项目都是怎么废的，及其影响。以及对 ShokaX 未来的展望

## 四个项目

### valaxy-theme-shokaX

Valaxy Theme ShokaX 是 ShokaX 向 Valaxy 迁移的第一次尝试，也是第一次向现代平台迁移的尝试

综合来看，这次迁移几乎是必然失败的：

- Valaxy 太年轻了。2023年的 Valaxy 仍然处于早期开发阶段，社区高度不成熟，文档存在严重缺陷
- 技术栈问题。当时的 ShokaX 处在 0.2->0.3过渡期，技术栈高度落后过时，直接将主题迁移到 Vue 3 上工作量大且难以完成
- 整体思路问题。主要维护者（也就是我）当时的主要思路是保持多旗舰，即 Valaxy + Nuxt + Hexo 并行，事实证明完全不可行且浪费大量资源
- 人员能力问题。其中一名主要维护者（我）先前几乎未接触过现代前端，未学习过 Vue 框架及相关内容，导致迁移进展缓慢

所以，因为进展缓慢和 SMB 路线的出现，valaxy-theme-shokaX 很快便终止开发了，其遗产（几乎没有）转移到了astro-theme-shokaX中

### astro-theme-shokaX

astro-theme-shokaX是一个无疾而终的项目，因为其没有面临像 Valaxy-theme-shokaX 那样的严重问题，其只面临**工作量大**这一个问题

当时由于 AI Coding 尚未成熟，模板间的翻译工作仍主要依靠人力进行，导致了严重的进展缓慢问题

ShokaX 本身逻辑复杂，代码老旧，人为理解后重新实现就相当困难，再加上 ShokaX 本身包含大量细节，而我在迁移时过于注重细节，导致总体进度受阻，整体开发周期超期

最终在2024年6月前后，虽然没有任何公告说明仓库存档和终止开发，但 astro-theme-shokaX 停止了开发。我忘了当时终止开发的原因了，但我认为和 SMB 路线的推进有关

### Hexo ShokaX SMB

这是一个很缝合怪的路线。本质是在 Hexo 的传统引擎中强行插入一套基于 esbuild 的半现代构建系统，并依赖这个系统进行现代化迁移

这个系统在 ShokaX 中起到了相当大的作用，支撑了 0.3/0.4 模块化和0.5 UIKit 两次大版本更新，但是：

- 其未能解决模板渲染问题。这导致了 SMB 路线的 ShokaX 事实上未进行任何组件迁移，并且一定程度上拖后了现代化迁移的进行
- 其仍然基于 Hexo。SMB 系统为了和 Hexo 兼容付出了相当大的代价
- 其未能解决根本问题。ShokaX 事实上并不需要一个 esbuild，而需要一个基于 Vite 的真正现代化平台

SMB 在 Hexo ShokaX 中仍在使用，但已事实性终止维护
SMB 路线降低了现代化迁移的紧迫性，但为 UIKit 的发展提供了机会

### ShokaX UIKit

ShokaX 向 Valaxy 迁移的第二次尝试，仍然遇到了和之前一样的问题——**Valaxy 不是一个理想的平台**

目前因为 UIKit 尚未完全终止开发，我无法做出全面的评价。但我认为 Valaxy 很难称之为是 Hexo 的理想继任者，Astro 更适合担任这个工作。

UIKit 产出了一些 Vue 3 组件和 Nyx-player，这些组件被转译到 Svelte 后被用于 ShokaX Astro

## 有关分析

### 为什么说“Valaxy 不是一个理想的平台”

我长期以来是高度支持 Valaxy 的，否则我也不会前后两次尝试迁移到 Valaxy。

Valaxy 的主要问题其实是缺少一个合理的生态位，导致在 Astro 出现之后其逐渐不是一个“理想的平台”了。在2023年的时候，Astro 和 Valaxy 同样不成熟，Valaxy 作为一个专攻博客方向的框架是值得使用的。

但现在是2025年，2年过去了，Valaxy 现在的生态和文档并未有明显的进步，而 Astro 已经是一个相当成熟的解决方案了。这种情况下，Valaxy 事实上就已经失去了它的生态位，毕竟 Astro 上已经有成熟解决方案了，而且扩展性可靠性更强，我为什么不用呢？
